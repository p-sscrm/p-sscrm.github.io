P.4.4 Executable security testing

Task Details

Objective

Discover vulnerabilities only detected through executable testing

Definition

Executable testing, such as done by in-house testers or third-party penetration testers, is performed to find previously-undiscovered vulnerabilities. In-house testers should be guided by design review results, threat modeling, security requirements, and the security mechanisms in security features. Testing should include containers. Automate security tests as possible. Scope the testing, design the tests, perform the testing, and document the results, including recording and triaging all discovered issues and recommended remediations in the development teamâ€™s workflow or issue tracking system. Test the build process and scripts.

Assessment Questions

  1. How are security features tested?
  2. How are test cases developed to test the security of an application?
  3. Is security testing factored into a product's test suite?
  4. Is executable code testing, such as in-house or external penetration testing, used to find vulnerabilities not found by previous reviews, analysis, and testing?
  5. Do you do executable testing on containers?
  6. Do you test your build process, scripts, GitHub actions?
