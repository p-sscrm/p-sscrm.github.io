G.1.3 Produce attestation

Task Details

Objective

Produce evidence of the use of secure software development practices.

Definition

Configure tools to generate artifacts to create an audit trail of the use of secure software development practices that conforms with record retention requirements and preserves the integrity of the findings and the confidentiality of the information. Assign responsibility for creating artifacts that tools cannot generate. Attestation should be immutable and published in the source repository releases, in the package registry, or elsewhere with their existence in a transparency log.

Assessment Questions

  1. Can you tell me about automated or manual processes for producing artifacts that attest to the use of secure development practices?
  2. Where are the artifacts stored?
  3. What are your auditability/retention requirements for these artifacts?
  4. Is responsibility assigned for creating needed artifacts that tools cannot generate?
  5. Do you use a framework, like in-toto, to produce authenticated meta-data about artifacts, such as for attestation?
  6. Do you need to provide self-attestation for your product?
  7. Is the attestation immutable and published in the source repository releases, in the package registry, or elsewhere with their existence in a transparency log?
