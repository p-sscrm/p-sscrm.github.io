G.1.5 Deliver SBOM

Task Details

Objective

By providing SBOM, enable internal and external customers to analyze the contents of the final software package, including the version of the dependencies.

Definition

Generate (preferably during build time) and provide a SBOM (in a machine-readable, NTIA-supported format) for generated products. SBOMs should be digitally signed using a verifiable and trusted key. VEX data for all components should also be considered.

Assessment Questions

  1. Do you deliver an SBOM and provenance data that satisfy your contracts with software acquirers, such as in a standard-base format?
  2. How and when is the SBOM built?
  3. What tool do you use?
  4. Do you digitally sign the SBOMs?
  5. Do you produce a VEX supplement for your SBOM?
  6. What is your process for generating the VEX information?
